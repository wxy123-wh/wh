# 电商产品口碑监测与迭代优化辅助系统（蓝牙耳机）— MVP 需求范围（强约束）

## 1. 目标（必须实现）
系统把“电商评论”自动转成“维度化口碑指标 + 优先级问题列表 + 可视化仪表盘”，用于支持产品迭代决策。

MVP 必须满足：
1) 支持 CSV 导入评论数据（批量入库、去重、清洗）；
2) 自动做维度归因（音质/续航/降噪/连接/佩戴/麦克风/外观/性价比）；
3) 自动做情感分析（POS/NEU/NEG + score）；
4) 自动做关键词 TopN（支持按维度统计）；
5) 自动输出“问题优先级 Top 列表”（维度级 + 关键词级）；
6) 前端三页可用：总览、维度分析、评论检索/详情；
7) 导入后自动触发分析流水线，分析结果可被前端查询展示。

## 2. 明确不做（MVP 禁止实现）
- 不做爬虫/多平台自动采集（只做 CSV 导入）。
- 不做预警中心/竞品对比/营销建议生成/LLM 文本生成。
- 不做复杂登录权限系统（不做账号体系；开发环境直接访问 API）。
- 不做深度学习模型训练与在线推理服务（只做可解释的规则/词典/统计方法）。

## 3. 用户与核心使用流程（必须跑通）
角色：产品经理（唯一角色）

流程：
1) 导入 CSV 评论数据；
2) 打开总览页：看到总评论量、负向率、负向趋势、Top 问题优先级；
3) 打开维度分析：点击维度看到该维度趋势与 Top 关键词；
4) 打开评论页：按维度/情感筛选评论，打开详情看到“命中关键词 + 维度情感”。

## 4. 模块拆分（必须按模块实现）
A. 数据接入模块（Ingestion）
- CSV 导入
- 去重、校验、入库

B. 清洗与分析模块（Processing + Analysis）
- 清洗（去 HTML/规范空白/过滤噪声）
- 维度归因（词典命中）
- 情感分析（情感词典 + 否定词规则）
- 关键词统计（频次 + NEG 频次）

C. 决策支持模块（Decision）
- 问题优先级计算（统一公式）
- 输出 TopN（维度级 + 关键词级）

D. 展示模块（Dashboard）
- 总览页
- 维度分析页
- 评论列表/详情页

## 5. 技术栈（固定）
- 后端：Spring Boot（REST API）+ MySQL 8.x
- 前端：Vue3 + Element Plus + ECharts
- 文本处理：Java 实现规则/词典/统计（不引入训练流程）

## 6. 项目结构（强约束）
/docs  放需求文件
/data  放 aspects.json、sentiment_lexicon.json、stopwords.txt、sample_reviews.csv
/backend Spring Boot 工程
/frontend Vue 工程

## 7. 交付验收（必须）
- 后端：可启动、建表脚本可执行、导入可用、API 全部可用
- 前端：三页可用且能展示真实数据
- docs/05 的验收用例全部通过
